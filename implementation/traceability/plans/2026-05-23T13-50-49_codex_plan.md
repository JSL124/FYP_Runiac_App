4. Backward compatibility plan

Use copy-first migration. Keep legacy `tools/agent-review/prompts/runiac/` and `tools/agent-review/config/runiac.agent-review.env.example` in place during the first batch.

Runner should default to:

```text
AGENT_REVIEW_PROFILE=runiac
AGENT_REVIEW_PROFILE_DIR=tools/agent-review/profiles/$AGENT_REVIEW_PROFILE
```

Prompt defaults should resolve from the selected profile. If the new profile files are missing, the runner may fall back to current legacy Runiac paths for one transition period.

5. Runiac profile preservation plan

The Runiac profile must retain these rules in its prompts and README:

- `docs/submissions/pdd/` is the frozen submitted PDD baseline.
- `docs/pdd/` is internal working/reference material only.
- Flutter owns UI, navigation, GPS tracking UI, local interaction, and client integration.
- Firebase Authentication owns identity.
- Cloud Firestore stores app data.
- Cloud Functions own trusted XP, streak, level, rank, weekly XP, monthly XP, leaderboard score, validation, aggregation, entitlement, and expert-plan governance logic.
- `subscriptionStatus` controls Basic/Premium access.
- `userRole` controls Platform Administrator and Medical Trainer/Expert governance access.
- Medical Trainer/Expert cannot publish expert plans directly.
- Premium must not create XP, rank, leaderboard score, or competitive advantages.
- no secrets/API keys.
- no Flutter/Firebase scaffold unless explicitly approved.

6. CLAUDE.md handling

Keep root `CLAUDE.md` in place for now because the current Claude command appends it directly. In the first batch, document that `CLAUDE.md` is the Runiac project-level Claude system prompt. Later, consider profile-specific Claude append prompts only after one more successful migrated workflow run.

7. Runner config/profile loading plan

Update `run_plan_review.sh` to support:

- `AGENT_REVIEW_PROFILE`, default `runiac`
- `AGENT_REVIEW_PROFILE_DIR`, default `tools/agent-review/profiles/$AGENT_REVIEW_PROFILE`
- `AGENT_REVIEW_CONFIG`, still supported as an override
- prompt defaults based on `$AGENT_REVIEW_PROFILE_DIR/prompts/`
- dry-run default preserved as `DRY_RUN=1`

Do not change the tested safety commands:

- Codex plan/decision: `--sandbox read-only --ask-for-approval never -C . exec`
- Claude review: `--permission-mode plan --tools "Read,Grep,Glob"`
- Codex implementation: `--sandbox workspace-write --ask-for-approval on-request -C .`
