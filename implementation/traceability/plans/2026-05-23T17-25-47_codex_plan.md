## Proposed Plan

1. Executive recommendation  
   Implement a minimal compatibility-first efficiency batch. Do not add Codex-as-reviewer, conditional decision skipping, silent fallback, or cost logging in this batch.

2. Efficiency batch design  
   Add two config variables:
   - `REVIEW_PROVIDER=claude|none`, default `claude`
   - `REVIEW_MODE=standard|lite`, default `standard`

   Valid combinations:
   - `claude + standard`: current behavior.
   - `claude + lite`: Claude review with shorter Runiac lite contract.
   - `none + standard`: manual checklist artifact/output, no Claude call.
   - `none + lite`: same no-Claude manual checklist path; optionally use concise checklist language.

3. Proposed config variables and defaults  
   Add defaults in `run_plan_review.sh` and document them in the Runiac env example:

   ```bash
   REVIEW_PROVIDER="${REVIEW_PROVIDER:-claude}"
   REVIEW_MODE="${REVIEW_MODE:-standard}"
   REVIEW_PROMPT_LITE="${REVIEW_PROMPT_LITE:-$AGENT_REVIEW_PROFILE_DIR/prompts/07_review_contract_lite.md}"
   MANUAL_REVIEW_CHECKLIST_PROMPT="${MANUAL_REVIEW_CHECKLIST_PROMPT:-$AGENT_REVIEW_PROFILE_DIR/prompts/06_manual_review_checklist.md}"
   ```

4. Proposed runner changes  
   Update `usage()` to document the new variables. Add validation helpers that reject unknown values. Resolve the new profile prompt paths using existing profile path compatibility patterns.

   Adjust `require_common_paths` so:
   - standard Claude mode requires `02_claude_review_plan.md`
   - lite Claude mode requires `07_review_contract_lite.md`
   - provider `none` requires `06_manual_review_checklist.md`
   - decision and implementation prompts remain required as today

   Add a `run_manual_review_step` that writes or prints a clearly labelled manual checklist artifact. It must say no automated Claude review occurred.

   Update `cmd_review` and `cmd_pipeline` to branch on `REVIEW_PROVIDER`. For `none`, skip `require_agent_commands_for_actual_run 0 1` and never call `claude`.

   Keep `run_decision_step`, but make the decision input label accurate:
   - `Claude review:` only when Claude ran.
   - `Manual checklist / no automated review:` when provider is `none`.

5. Proposed Runiac profile prompt files  
   Add:
   - `tools/agent-review/profiles/runiac/prompts/06_manual_review_checklist.md`
   - `tools/agent-review/profiles/runiac/prompts/07_review_contract_lite.md`

   The manual checklist should cover AGENTS/PRD/submitted PDD consistency, untouched paths, Runiac backend ownership rules, safety restrictions, and approval gates.

   The lite review contract should ask Claude for concise findings only: blockers, high-risk omissions, unsafe operations, and one final recommendation.

6. Manual checklist mode  
   `REVIEW_PROVIDER=none` must not call Claude. It should produce a review artifact path under `implementation/traceability/reviews/` or print the dry-run equivalent. Suggested artifact name should avoid claiming Claude, for example:

   ```text
   <timestamp>_manual_review_checklist.md
   ```

   Required label inside output:

   ```text
   Manual checklist only. No Claude or automated external reviewer was used.
   ```

7. Lite review mode  
   `REVIEW_MODE=lite` with `REVIEW_PROVIDER=claude` should still use Claude with:
   - `--permission-mode plan`
   - `--tools "Read,Grep,Glob"`
   - `--append-system-prompt "$(cat CLAUDE.md)"`

   It should only swap the review prompt to `07_review_contract_lite.md`. It must not loosen any tool or permission restrictions.

8. Backward compatibility plan  
   Defaults preserve current behavior. Existing `plan`, `review`, `decision`, `implement`, and `pipeline` subcommands remain. Existing env files without new variables still run standard Claude review. Existing explicit `REVIEW_PROMPT` overrides should continue to work for standard mode.

9. Failure handling  
   Invalid `REVIEW_PROVIDER` or `REVIEW_MODE` should fail before creating artifacts. Missing lite/manual prompt files should fail with `required file not found`. Claude failure in `REVIEW_PROVIDER=claude` must stop the pipeline; no silent fallback. Manual mode must never describe the manual checklist as a review by Claude.

10. Acceptance checks  
   Since this is shell/docs-only, future approved implementation can use read-only/dry-run checks, not Flutter/Firebase/npm/tests/builds:
   - `bash -n tools/agent-review/runner/run_plan_review.sh`
   - `DRY_RUN=1 TASK_PROMPT="Smoke test only. Do not modify files." tools/agent-review/runner/run_plan_review.sh pipeline`
   - same dry-run with `REVIEW_MODE=lite`
   - same dry-run with `REVIEW_PROVIDER=none`
   - inspect output text to confirm Claude is absent in provider `none`
   - `grep -R "REVIEW_PROVIDER\\|REVIEW_MODE" -n tools/agent-review`

11. Suggested first approved implementation scope  
   Only update the generic runner, Runiac profile prompt files, and workflow documentation. Do not touch PDD, PRD, Flutter, Firebase, tests, or traceability records.
