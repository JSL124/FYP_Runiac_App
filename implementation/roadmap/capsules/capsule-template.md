# Capsule Template

## Parent Phase

`implementation/roadmap/phases/<phase-file>.md`

## Goal

State the single implementation or review cycle this capsule will complete.

## Allowed Scope

- List the exact work allowed for this capsule.
- Keep scope phase-bound and short-lived.

## Forbidden Scope

- List all nearby work that must not be touched.
- Include tier restrictions and any blocked external systems.

## Exact Target Files

- `path/to/file`

## Required Tests

- List required tests or state why tests are not applicable.

## Required Validation

- List required review/checker passes.
- Include gate or ADR checks where relevant.

## Required Evidence

- List command outputs, review notes, screenshots, or traceability artifacts required before exit.

## Rollback Conditions

- Define what requires reverting or stopping the capsule.

## Exit Criteria

- [ ] Target files completed.
- [ ] Required tests or validation completed.
- [ ] Required evidence recorded.
- [ ] Snapshot updated if state changed.
- [ ] CURRENT.md updated if active capsule, phase, gate status, or forbidden scope changed.
